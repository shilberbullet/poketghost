# 포켓요괴 (PoketGhost)

## 게임 소개
포켓요괴는 한국의 전통 요괴들을 수집하고 육성하는 턴제 RPG 게임입니다. 플레이어는 요괴들을 모아 전투를 벌이고, 아이템을 사용하여 요괴를 회복시키며, 다양한 스테이지를 클리어해 나가는 모험을 즐길 수 있습니다.

## 주요 기능

### 1. 요괴 시스템
- 다양한 한국 전통 요괴 수집 및 육성
- 각 요괴마다 고유한 특성과 능력치
- 레벨업 시스템을 통한 요괴 성장
- 파티 시스템을 통한 최대 5마리의 요괴 관리

### 2. 전투 시스템
- 턴제 전투 방식
- 요괴별 고유 기술 사용
- 상성 시스템을 통한 전략적 전투
- 기절 상태의 시각적 표시 (빨간색으로 표시)

### 3. 아이템 시스템
- 다양한 회복 아이템 사용
- 기절한 요괴에 대한 아이템 사용 제한
  - 탕국, 막걸리: 기절한 요괴에게 사용 가능
  - 제삿밥, 식혜: 기절한 요괴에게 사용 불가
- 인벤토리 시스템을 통한 아이템 관리

### 4. 스테이지 시스템
- 일반 스테이지와 보스 스테이지
- 5의 배수 스테이지마다 자동 저장
- 다음 스테이지 요괴 정보 저장
- 시간대별 다른 요괴 출현

### 5. 저장 시스템
- 수동 저장 기능
- 자동 저장 기능 (5의 배수 스테이지)
- 단일 저장 파일 관리

### 6. 게임 설정
- 텍스트 속도 조절
- 텍스트 표시 모드 선택
- 상성 힌트 표시 설정
- 디버그 모드

### 지역 시스템
- 게임 시작 시 경상도/전라도 중 한 곳을 선택하여 모험 시작
- 각 지역은 실제 한반도 지도를 기반으로 연결되어 있음
- 10스테이지마다 방문하지 않은 인접 지역 중 한 곳으로 자동 이동
- 이미 방문한 지역은 다시 방문하지 않음(저장/불러오기 시에도 유지)
- 지역 이동 시 연결된 지역 목록이 안내됨
- 기존 '초보자의 숲' 등 하드코딩 지역명 완전 제거, 실제 지역명만 사용
- 맵 보기 기능은 아직 구현되지 않음

## 게임 시스템 상세 설명

### 요괴 시스템
- **요괴 등급**
  - 일반: 기본 능력치, 1-2개의 특수 기술
  - 희귀: 높은 능력치, 2-3개의 특수 기술
  - 전설: 최고 능력치, 3-4개의 특수 기술
- **요괴 특성**
  - 각 요괴마다 고유한 특성 보유
  - 전투 중 특정 조건에서 발동
  - 능력치 상승, 상태이상 회복 등 다양한 효과
- **요괴 육성**
  - 전투 승리 시 경험치 획득 (기본 경험치: 레벨당 80)
  - 레벨업 시 HP가 초기화되지 않고 증가량만큼만 추가
  - 레벨업 시 남은 경험치 유지
  - 기술 습득 및 PP 증가
- **최대 동료 수**
  - 기본 5마리까지 동료 요괴 보유 가능
### 전투 시스템
- **턴제 전투**
  - 스피드에 따른 행동 순서 결정
  - 각 턴마다 1개의 행동만 가능
  - 기술 사용, 아이템 사용, 도망 선택 가능
- **기술 시스템**
  - 각 요괴마다 4가지 기술 보유
  - 기술별 고유한 위력과 효과
  - 명중률에 따른 성공/실패 판정
- **PP 시스템**
  - 기술 사용 시 PP 소모
  - PP가 0이면 해당 기술 사용 불가
  - 회복 아이템으로 PP 복구 가능
- **도망 시스템**
  - 일반 스테이지에서 50% 확률로 도망 가능
  - 보스 스테이지에서는 도망 불가
  - 도망 실패 시 적의 공격 기회 제공
- **보스 스테이지**
  - 10스테이지마다 등장하는 강력한 요괴
  - 도망 불가, 부적 사용 불가
  - 승리 시 특별한 보상 제공

### 아이템 시스템
- **아이템 등급**
  - 일반: 기본적인 효과, 높은 획득 확률
  - 희귀: 강력한 효과, 낮은 획득 확률
  - 초희귀: 최고급 효과, 매우 낮은 획득 확률
- **아이템 종류**
  - 부적: 요괴 포획에 사용, 등급별 성공률 차이
  - 회복 아이템: HP, PP 회복, 상태이상 치료
  - 강화 아이템: 능력치 상승, 특수 효과 부여
- **아이템 제한**
  - 일반 아이템: 최대 99개 보유 가능
  - 특수 아이템 (작두, 무당방울): 최대 5개 보유 가능
  - 인벤토리 슬롯: 최대 20개
- **아이템 보상 시스템**
  - 전투 승리 시 3개의 랜덤 아이템 중 선택
  - 아이템 등급에 따른 출현 확률 조정
  - 아이템 목록 초기화 비용이 스테이지 구간에 따라 다르게 적용됩니다.
    - 1~10스테이지: 100전
    - 11~20스테이지: 200전
    - 각 구간 내에서 초기화할 때마다 2배씩 증가
    - 게임 저장/불러오기 시에도 스테이지에 맞는 비용이 유지됩니다.
- **식혜(회복 아이템) 시스템 모듈화**
  - 식혜 관련 코드는 별도 모듈(`src/sikhye_system.c`, `include/sikhye_system.h`)로 분리되어 유지보수 및 확장성이 향상되었습니다.

### 스테이지 시스템
- **스테이지 진행**
  - 1~100스테이지까지 진행 가능
  - 스테이지별 다른 요괴 출현
  - 난이도 점진적 상승
- **보상 시스템**
  - 기본 보상: 100전
  - 스테이지 보너스: 10스테이지마다 50전씩 증가
  - 랜덤 보너스: 0~49전
  - 10의 배수 스테이지: 보상 2배
  - 보스 스테이지: 특별 보상 추가
- **지역 시스템**
  - 스테이지별로 다른 지역과 지형 설정
  - 지역별 특색 있는 요괴 출현
  - 지형에 따른 전투 효과 변화

### 저장 시스템
- **자동 저장**
  - 스테이지 클리어 시 자동 저장
  - 저장 실패 시 경고 메시지 출력
- **수동 저장**
  - 메뉴에서 언제든지 저장 가능
- **저장 데이터**
  - 현재 스테이지 정보
  - 보유 금액
  - 동료 요괴 정보
  - 인벤토리 아이템
  - 게임 설정

## 실행 방법

### 1. 개발 환경 준비
1. MinGW 설치 (C 컴파일러)
   - [MinGW 다운로드 페이지](https://sourceforge.net/projects/mingw/)에서 설치
   - 설치 시 `mingw32-base`와 `mingw32-gcc-g++` 패키지 선택
   - 설치 후 시스템 환경 변수 PATH에 MinGW의 bin 폴더 경로 추가
     (예: `C:\MinGW\bin`)

2. Git 설치
   - [Git 다운로드 페이지](https://git-scm.com/downloads)에서 설치

### 2. 프로젝트 다운로드
1. 명령 프롬프트(cmd) 실행
   - Windows 키 + R을 누르고 'cmd' 입력 후 실행

2. 프로젝트 폴더 생성 및 이동
#### 원하는 위치로 이동 (예: C 드라이브)
```bash
cd C:\
```

#### 프로젝트 폴더 생성
```
mkdir poketghost
```

#### 생성한 폴더로 이동
```
cd poketghost
```

#### GitHub 저장소 클론
```
git clone https://github.com/shilberbullet/poketghost.git -b codex/refactor-global-state-management
```

### 3. 컴파일 및 실행

#### 소스 코드 컴파일
```bash
gcc -o poketghost.exe src/*.c core/state.c -I include -I core -lm
```
#### 게임 실행
```
poketghost.exe
```

### 2. 모듈별 주요 기능
- **main.c**: 게임 초기화, 메인 루프, 종료 처리
- **menu.c**: 메인 메뉴, 전투 메뉴, 설정 메뉴 등 UI 관리
- **game.c**: 게임 상태, 진행 상황, 이벤트 처리
- **input.c**: 키보드 입력 처리, 입력 유효성 검사
- **text.c**: 텍스트 출력, 대화창, 메시지 처리
- **stage.c**: 스테이지 생성, 진행, 보상 처리
- **yokai.c**: 요괴 생성, 능력치, 특성 관리
- **move.c**: 기술 시스템, PP 관리, 효과 처리
- **battle.c**: 전투 로직, 데미지 계산, 상태이상
- **party.c**: 파티 관리, 요괴 교체, 순서 변경
- **savefile.c**: 저장/불러오기, 데이터 직렬화
- **item.c**: 아이템 시스템, 인벤토리 관리
- **sikhye_system.c**: 식혜 아이템 시스템, PP 회복 처리
- **heal_system.c**: 회복 시스템, HP/PP 회복 처리
- **reward.c**: 전투 보상, 아이템 선택 시스템
- **escape.c**: 도망 시스템, 전투 이탈 처리

1. **코드 재사용성**

2. **유지보수성**
   - 버그 수정이 특정 모듈에 국한됨

4. **확장성**
